CsHookInterface.h : Application과 CS System의 연동을 위한 Hook Interface 정의 파일 입니다.

App: DirectXContainer.exe, GolfzonVision.exe와 같은 CS Sytem과 연동 하려는 Application을 의미합니다. Hook Dll: CS(CloudStreaming)과 App을 연동하기 위해 Inject 하는 Dll을 의미합니다.

CS System간 연동을 위한 App과 Hook Dll 간의 통신 흐름을 설명합니다.

A. Hook Dll은 App으로 Inject 되고, App의 Window Handle을 얻어서 WM_CS_MSG_START를 통해서 App/Hook Dll 연동의 시작을 알려줍니다. B. App의 Window Message 처리부 (WndProc()) 에서 WM_CS_MSG_START를 받습니다. 이후 아래 메세지들을 통해서 SendCallback, InfoCallback, RecvCallback의 포인터를 전달해 줍니다. WM_CS_CALLBACK_SEND WM_CS_CALLBACK_INFO WM_CS_CALLBACK_RECV

  • SendCallback() 함수는 App의 데이터를 STB으로 전달하기 위해 호출할 수 있습니다.
  • RecvCallback() 함수는 App으로 STB의 데이터를 전달해 줍니다.
  • InfoCallback() 함수는 CS 연동 데이터를 얻기 위해 호출할 수 있습니다. Xml 형식으로 데이터를 얻을 수 있으며 Xml 버퍼의 최대 크기는 CS_MAX_XML_SIZE (4k) 입니다. 여기에서 <MacAdress><ClientId> 등을 얻을 수 있습니다.

App과 Hook Dll의 연동 메세지를 정의합니다.

WM_CS_MSG_START: App과 Hook Dll의 연동 시작을 알려주며, wParam으로 CSHookInterface의 포인터를 전달합니다. WM_CS_CALLBACK_SEND: SendCallback의 포인터를 전달해 줍니다. WM_CS_CALLBACK_RECV: RecvCallback의 포인터를 전달해 줍니다. WM_CS_CALLBACK_INFO: InfoCallback의 포인터를 전달해 줍니다. WM_CS_MSG_END: App과 Hook Dll의 연동 종료를 알려주는 메세지 입니다.
